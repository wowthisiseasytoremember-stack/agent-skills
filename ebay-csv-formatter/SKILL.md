# eBay File Exchange CSV Format Rules and Processing

This skill generates and processes CSV files formatted according to eBay's File Exchange specifications for various listing operations.

## Input Method

This skill can now process input data directly from a specified CSV file path. When invoking the skill, provide the path to your input CSV file. The skill will then parse this file according to the rules and perform the requested operation.

## Operations Supported

The skill supports the following operations: Ending, Revising, and Adding new listings.

## CSV Format Rules

The skill adheres to these rules for generating CSV output:

### 1. Ending Listings

#### Header Row (EXACT format needed):
```csv
Action(SiteID=US|Country=US|Currency=USD|Version=1193|CC=UTF-8),ItemID,EndCode
```

#### Data Row Format:
```csv
End,ItemID,EndCode
```

#### Valid `EndCode` Values:
*   `NotAvailable` - Item is no longer available for sale
*   `LostOrBroken` - Item was lost or broken
*   `Incorrect` - Listing has an error (e.g., wrong price or description)
*   `OtherListingError` - Other listing error

#### Example:
```csv
Action(SiteID=US|Country=US|Currency=USD|Version=1193|CC=UTF-8),ItemID,EndCode
End,297503767193,NotAvailable
End,297512715740,NotAvailable
```

---

### 2. Revising Listings

#### Header Row:
```csv
Action(SiteID=US|Country=US|Currency=USD|Version=1193|CC=UTF-8),ItemID,Title,StartPrice,Quantity,Description
```

#### Data Row Format:
```csv
Revise,ItemID,New Title,19.99,5,New description text
```

---

### 3. Adding New Listings

#### Header Row:
```csv
Action(SiteID=US|Country=US|Currency=USD|Version=1193|CC=UTF-8),Title,StartPrice,Quantity,Category,Description,PicURL
```

#### Data Row Format:
```csv
Add,Product Title,29.99,10,12345,Product description,http://example.com/image.jpg
```

## Universal CSV Rules

Adhere to these rules for all CSV files processed or generated by this skill:

1.  **NO quotes around any values** (unless the value itself contains a comma).
2.  **NO BOM** (Byte Order Mark) at the start of the file. Save as UTF-8 without BOM.
3.  **Use commas as delimiters** only.
4.  **NO spaces after commas** (unless part of the value).
5.  **Save with UTF-8 encoding.**
6.  **Use Windows line endings** (CRLF) if on Windows.
7.  The **first row MUST be the header** with the `Action` column specified correctly.
8.  **Action column format:** `Action(SiteID=US|Country=US|Currency=USD|Version=1193|CC=UTF-8)`.

## Common Column Names:

*   `Action`: Add, Revise, End, Relist, VerifyAdd
*   `ItemID`: eBay item number
*   `CustomLabel`: Your SKU/custom identifier
*   `Title`: Listing title (80 chars max)
*   `StartPrice`: Starting/Buy It Now price
*   `Quantity`: Number available
*   `Category`: eBay category ID
*   `Description`: HTML allowed
*   `PicURL`: Image URL (pipe-separated for multiple: url1|url2|url3)
*   `EndCode`: Reason for ending (see above: `NotAvailable`, `LostOrBroken`, `Incorrect`, `OtherListingError`)
*   `Duration`: Days_7, Days_10, GTC (Good Till Cancelled)
*   `Format`: FixedPrice, Auction, StoresFixedPrice
*   `Location`: City, State
*   `ShippingService-1:Option`: USPSPriority, FedEx2Day, etc.
*   `ShippingService-1:Cost`: Shipping cost
*   `ConditionID`: 1000=New, 3000=Used, etc.

## IDE Configuration Tips:

When working with these CSV files, configure your IDE or text editor to:

1.  Save as **UTF-8 without BOM**.
2.  Use **comma** as the delimiter.
3.  **Do not quote fields automatically**.
4.  Use **CRLF line endings** (if on Windows).
5.  **Do not add extra spaces** after commas.

Most IDEs/Excel might auto-quote everything. Use a text editor like Notepad++, VS Code, or a script to generate the CSV properly if needed.
